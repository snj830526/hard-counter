# 테스트 가이드

## 기본 빌드 검사

프로젝트 루트에서 다음 명령을 사용한다.

```bash
xcodebuild -quiet \
  -project HardCounter.xcodeproj \
  -scheme HardCounter \
  -destination 'platform=iOS Simulator,name=iPhone 17 Pro' \
  -configuration Debug \
  CODE_SIGNING_ALLOWED=NO \
  build
```

실제 iOS 대상 컴파일도 확인한다.

```bash
xcodebuild -quiet \
  -project HardCounter.xcodeproj \
  -scheme HardCounter \
  -destination 'generic/platform=iOS' \
  -configuration Debug \
  CODE_SIGNING_ALLOWED=NO \
  build
```

문서와 소스의 공백 오류는 다음으로 확인한다.

```bash
git diff --check
```

## 3D 파이터 스파이크

`feature/3d-fighter-spike` 브랜치는 기본적으로 저폴리곤 3D 파이터를 표시한다. 같은 코드와
전투 상태에서 기존 2D 파이터를 비교하려면 다음 실행 인자를 추가한다.

```bash
xcrun simctl launch <SIMULATOR_UDID> \
  com.soonispapa.HardCounter \
  --legacy-2d-fighters
```

3D 상태에서 기존 쇼케이스 인자를 함께 사용해 다음을 비교한다.

- `--motion-showcase`: 스트레이트·스매시·어퍼컷에서 골반, 가슴, 어깨, 주먹의 회전 순서
- `--sway-showcase`: 가까운 거리에서 좌우 슬립, 풀백과 전진 실패의 실루엣
- `--impact-showcase`: 기술별 피격 방향과 전신 반응
- `--footwork-showcase`: 8방향 셔플, 급반전, 정지 착지와 이동 중 펀치·스웨이 전환
- `--fatigue-showcase`: 저스태미너 가드, 호흡과 하체 무게 표현
- 일반 솔로 경기: 이동 중 양발의 앞뒤 깊이, 무릎 굽힘과 발바닥 접지
- 이동 쇼케이스: 펀치·스웨이보다 이동 스텝이 명확히 느리고, 지지발은 그림자 위에 남은 채 이동 발만 들렸다가 착지하는지 확인
- 일반 솔로 경기의 중·근거리: 선수의 가드와 발이 읽힐 만큼 크게 보이고, 거리가 가까워질 때 줌이 급격히 튀거나 두 선수가 화면 밖으로 잘리지 않는지 확인
- 타격 쇼케이스: 화면 가독성용 확대를 적용해도 장갑 궤적이 머리 또는 몸통 표면에 닿기 전에는 체력 감소와 타격 이펙트가 발생하지 않는지 확인

선택 캐릭터의 3D 모션 프로필을 CPU 기술 시연에 복제하려면 두 인자를 함께 사용한다.

```bash
xcrun simctl launch <SIMULATOR_UDID> \
  com.soonispapa.HardCounter \
  --motion-showcase \
  --fighter-style-showcase
```

JIN, MASON, LEO를 각각 선택해 같은 CPU 기술 시퀀스에서 가드 높이, 스탠스 깊이,
풋워크 바운스와 대표 기술 실루엣을 비교한다. 기술이 시작되면 `PWR`, `SPD`, `RNG`가
표시되어 선택 캐릭터의 전투 스타일 배율도 함께 확인할 수 있다. 이 인자는 Debug 시연의
CPU 프로필만 선택 캐릭터와 맞추며 일반 경기와 근거리 대전 규칙은 변경하지 않는다.

캐릭터 선택에서 JIN, MASON, LEO를 같은 조건으로 각각 실행해 다음 차이가 보이는지 확인한다.

- JIN: 중간 높이의 좌우 균형 가드와 보통 보폭, 보통 체형과 짧은 머리, 클래식 트렁크. 어퍼컷은 먼저 무릎에 체중을 싣고 타격 때 다리가 펴져야 한다.
- MASON: 앞으로 기운 낮은 중심, 깊은 무릎, 얼굴 가까이 모은 가드와 무거운 회복, 넓은 어깨·굵은 목·큰 글러브·두꺼운 트렁크. 스매시는 골반과 가슴 회전 폭이 가장 커야 한다.
- LEO: 뒤에 둔 높은 중심, 길게 내민 리드핸드와 높은 리어 가드, 긴 보폭, 큰 호흡과 넓은 스웨이, 가는 몸통·좁고 긴 머리·작은 글러브·긴 복싱화. 스트레이트는 세 선수 중 가장 길고 빠르게 회수되어야 한다.
- 피부와 장비 색상을 구분하기 어려운 거리에서도 세 캐릭터의 몸통·장갑·트렁크 실루엣이 서로 달라야 한다.
- 정면뿐 아니라 대각선과 측면에서도 헤어스타일과 몸의 부피 차이가 유지되어야 한다.
- 가까운 포스트와 먼 포스트 모두 머리나 복싱화가 3D 렌더 타일 경계에서 잘리지 않아야 한다.
- 발밑의 진한 접촉 그림자와 오른쪽 아래의 연한 투영 그림자가 구분되며, 선수의 깊이가 바뀌어도 과도한 검은 원판처럼 보이지 않아야 한다.
- 링의 차가운 상부 조명과 따뜻한 캔버스 조명이 캐릭터 표면에도 같은 방향과 색조로 느껴져야 한다.
- 어떤 캐릭터도 이동·스웨이·피격·KO 중 무릎이 뒤로 꺾이거나 발목이 과회전하지 않아야 한다.
- 정지 호흡과 스웨이 중 발바닥이 압축된 접촉 그림자에서 눈에 띄게 분리되지 않아야 하며, 시작 포스트와 중앙에서도 같은 간격을 유지해야 한다.
- 대표 기술은 JIN 어퍼컷, MASON 스매시, LEO 스트레이트로 설정한다. 모션 실루엣과 함께 각 캐릭터의 `FighterCombatStyle` 위력·속도·사거리·소모 배율이 적용되어야 한다.

옆을 보는 자세에서도 양쪽 팔과 다리가 겹쳐 사라지지 않는지, 캐릭터 회전 중 몸의 부피가
유지되는지 확인한다. 3D 렌더러는 표현만 담당하므로 2D/3D 비교에서 사거리, 피해량,
스태미너, CPU 판단과 근거리 대전 동기화 결과가 달라지면 안 된다.

## CPU 모션 쇼케이스

멀티터치 조합 기술은 시뮬레이터의 순차 클릭으로 검증하지 않는다. Debug 앱을 설치한 뒤
다음 실행 인자로 시작하면 CPU가 스트레이트, 스매시, 어퍼컷을 반복해서 시연한다.

```bash
xcrun simctl launch <SIMULATOR_UDID> com.soonispapa.HardCounter --motion-showcase
```

각 기술 이름이 화면에 표시되며 CPU 이동과 일반 랜덤 공격은 중지된다. 세 기술은 실제
전투 엔진에서 `스웨이 → 펀치` 상태 전환을 거치므로 입력 조합 이후의 모션을 같은 조건으로
비교할 수 있다. 손가락 동시 입력 자체의 인식은 아래 실제 기기 항목에서 별도로 확인한다.

스웨이 방향만 반복 비교할 때는 다음 실행 인자를 사용한다.

```bash
xcrun simctl launch <SIMULATOR_UDID> com.soonispapa.HardCounter --sway-showcase
```

CPU가 좌우 슬립, 풀백, 전진 실패를 순서대로 시연한다. 좌우 슬립의 머리 이동 방향과
골반·무릎의 반대 균형, 풀백의 거리, 전진 실패의 구분을 같은 카메라에서 확인한다.
선수가 좌상·우상·좌하·우하를 바라보는 구도에서도 상·하·좌·우 및 대각선 입력과 머리의
화면 이동 방향이 같아야 한다. 방향 변환은 캐릭터의 로컬 축이 아니라 최종 화면 벡터를
기준으로 확인하며, 발과 접촉 그림자는 링 좌표에 남아야 한다.

시간축 모션 클립을 기존 포즈 전환과 직접 비교할 때는 다음 인자를 사용한다.

```bash
xcrun simctl launch <SIMULATOR_UDID> com.soonispapa.HardCounter --motion-clip-showcase
```

CPU가 `LEGACY LEAD STRAIGHT`와 `CLIP REAR STRAIGHT`를 번갈아 시연한다. 새 클립에서는
준비 동작 뒤 골반·어깨·뒷주먹이 순차적으로 전진하는지, 타격과 회수 중 양발이 매트에서
미끄러지지 않는지, 스트레이트를 맞은 선수의 발은 남고 전신이 하나로 밀리는지 확인한다.
정지 구간에는 가드가 고정 그림처럼 멈추지 않고 작은 호흡과 체중 이동이 이어져야 한다.

3D 풋워크와 액션 전환을 반복 비교할 때는 다음 인자를 사용한다.

```bash
xcrun simctl launch <SIMULATOR_UDID> com.soonispapa.HardCounter --footwork-showcase
```

CPU가 상하좌우와 네 대각선을 이동하고 각 구간 끝에서 발을 디딘 채 정지한다. 첫 번째
가로 이동에는 `STEP INTO PUNCH`, 대각선 이동에는 `MOVE INTO SWAY`가 삽입된다. 이동
입력이 액션 시작과 함께 끊겨도 진행 중이던 발이 기본 자세로 튕기지 않고 착지를 마쳐야
하며, 펀치·스웨이 포즈에서도 골반과 무릎의 보행 위상이 잠시 이어져야 한다. 이 인자는
Debug CPU 표현만 제어하며 일반 CPU 판단과 근거리 대전 입력에는 영향을 주지 않는다.

동작 진폭을 조정한 경우 세 쇼케이스를 60fps로 녹화해 공격의 준비–타격–후속 동작,
스웨이의 이탈–정점–복귀, 풋워크의 하중–이동 발–착지 구간을 프레임 단위로 확인한다.
정지 프레임의 작은 흔들림만 커지고 골반·흉곽·머리의 위치 차이가 보이지 않으면 역동성이
개선된 것으로 판단하지 않는다.

저스태미너 자세를 정지 상태에서 비교할 때는 다음 인자를 사용한다.

```bash
xcrun simctl launch <SIMULATOR_UDID> com.soonispapa.HardCounter --fatigue-showcase
```

CPU 스태미너 표시와 3D 표현만 0으로 유지한다. 가드가 완전히 풀리지는 않되 팔이 조금
낮아지고, 중심과 무릎이 내려가며 큰 호흡이 보여야 한다. 일반 경기에서는 전투 엔진의
실제 스태미너 변경 이벤트가 같은 표현 계층으로 전달되며, 이 시연 인자는 피해량·속도
계산을 바꾸지 않는다.

기술별 적중과 피격 리액션을 반복 비교할 때는 근접 위치로 시작하는 다음 인자를 사용한다.

```bash
xcrun simctl launch <SIMULATOR_UDID> com.soonispapa.HardCounter --impact-showcase
```

CPU가 스트레이트, 스매시, 어퍼컷을 실제 전투 판정으로 적중시킨다. Debug 시연에서만
시작 간격을 좁히며 일반 경기의 시작 위치, 사거리와 히트박스는 바꾸지 않는다.

## 조작 회귀 테스트

### 캐릭터 전투 특성

- JIN의 카운터가 일반 캐릭터보다 빠르게 시작하고 36 전후의 피해를 주며, 호스트와 게스트 어느 쪽에서도 동일하게 성립하는지 확인한다.
- 카운터 기회를 사용한 펀치가 빗나가도 같은 카운터 기회를 두 번째 펀치에 다시 사용할 수 없어야 한다.
- JIN의 어퍼컷은 기본 14보다 적은 스태미너를 사용하지만 일반 어퍼컷의 사거리와 위력 규칙은 유지하는지 확인한다.
- MASON의 스매시는 일반 스매시보다 강하지만 조금 짧고, 스태미너를 약 18% 더 사용하며 회수가 약 14% 느린지 확인한다. 스트레이트도 일반 선수보다 약하고 느려야 한다.
- LEO의 스트레이트는 위력이 낮은 대신 시작과 회수가 빠르고 약 8% 멀리 닿는지 확인한다. 스매시와 어퍼컷은 위력이 낮고 회수·소모가 불리해야 한다.
- 기본 능력치는 JIN `100/60/100%`, MASON `106/50/88%`, LEO `92/68/112%`(체력/스태미너/이동 속도)이며, MASON이 내구·위력·사거리 우위를 동시에 갖지 않는지 확인한다.
- 근거리 대전에서 같은 입력·거리·스태미너 상태라면 호스트와 게스트가 같은 피해량과 기술 프로필을 계산하는지 확인한다.

### 이동

- D-pad 중앙, 가장자리, 원 바깥의 왼쪽 아래 조작 영역에서도 입력이 시작되는지 확인한다.
- 왼쪽 아래 영역의 서로 다른 위치에서 터치를 시작하면 해당 지점이 스틱 중심이 되는지 확인한다.
- 터치를 시작한 순간에는 이동하지 않고, 시작점에서 드래그한 뒤에만 이동하는지 확인한다.
- 상하좌우 및 대각선 이동이 모두 가능한지 확인한다.
- 스틱을 조금 밀었을 때는 천천히, 끝까지 밀었을 때는 최고 속도로 이동하는지 확인한다.
- 입력 시작과 정지에 짧은 가속·감속이 있고 급반전에는 체중 전환이 느껴지는지 확인한다.
- 스틱을 누르는 즉시 캐릭터와 스틱 표시가 반응하며, 가속이 조작을 방해할 만큼 늦지 않은지 확인한다.
- 오른쪽 이동 중 즉시 왼쪽으로 전환해도 입력이 끊기지 않는지 확인한다.
- 오른쪽을 유지하다 왼쪽으로 급반전한 프레임에 스웨이를 눌러도 이전 오른쪽 방향이 아니라 최신 왼쪽 방향으로 상체가 움직이는지 확인한다.
- 이동 손가락이 패드 바깥으로 나가도 터치를 놓기 전까지 방향이 갱신되는지 확인한다.
- 이동 영역에 다른 손가락이 추가로 닿아도 기존 이동 방향이 상쇄되거나 끊기지 않는지 확인한다.
- 터치를 놓았을 때 짧게 감속하고 멈추는지 확인한다.
- 전진·후퇴·좌우 이동에서 진행 방향 쪽 발이 먼저 짧게 움직이고 반대 발이 따라오는지 확인한다.
- 전진에서는 앞발, 후퇴에서는 뒷발이 먼저 움직이며 첫 발이 착지한 뒤 반대 발이 스탠스 폭을 회복하는지 확인한다.
- 선수 전체의 화면 이동과 별개로 발과 무릎의 상대 위치 변화가 눈에 보이며 미끄러지는 실루엣으로 남지 않는지 확인한다.
- 양발이 보행처럼 크게 교차하거나 번갈아 높이 들리지 않고 기본 스탠스 폭을 유지하는지 확인한다.
- 정지 자세에서 양쪽 허벅지가 바깥으로 벌어지고, 무릎 아래는 다시 중심 쪽으로 향해 다리가 교차하지 않는지 확인한다.
- 이동발 무릎은 부드럽게 접히고 지지발 쪽 골반은 작게 반응해 다리가 한 막대처럼 보이지 않는지 확인한다.
- 이동 입력을 놓거나 반대 방향으로 급전환해도 진행 중인 발이 즉시 기본 자세로 튕기지 않고 착지 단계를 마치는지 확인한다.
- 보폭과 무릎 각도가 바뀌어도 지지하는 발바닥이 매트에서 과하게 들리거나 정강이와 함께 뻣뻣하게 회전하지 않는지 확인한다.
- 정지하면 가드 자세로 자연스럽게 돌아오며 미세한 호흡이 이어지는지 확인한다.
- JIN은 발과 가드가 안정적으로 함께 움직이고, MASON은 느린 발 교대와 짧고 낮은 보폭으로 따라붙으며, LEO는 빠른 발 교대·긴 보폭·큰 상체 리듬을 보이는지 같은 입력으로 비교한다.

### 멀티터치

- 이동을 유지한 채 `PUNCH`가 동작하는지 확인한다.
- `PUNCH`와 `SWAY`를 누르는 순간 버튼 피드백과 동작 준비가 시작되는지 확인한다.
- 이동 방향을 급반전하면서 `SWAY`를 눌러도 새 방향이 사용되는지 확인한다.
- 스틱을 급히 꺾는 프레임에 `SWAY`를 눌러도 직전 방향이 아니라 버튼을 누른 순간의 방향으로 몸이 이동하는지 확인한다.
- 액션 버튼을 놓아도 이동 터치가 유지되는지 확인한다.
- 시스템 제스처나 앱 전환 후 입력이 고정된 채 남지 않는지 확인한다.

### 펀치와 거리

- 리드와 리어 손이 번갈아 나오는지 확인한다.
- 리드 잽은 빠르고 짧게, 리어 스트레이트는 몸통과 뒷발이 함께 회전하는지 확인한다.
- 펀치에서 골반이 먼저 회전하고 어깨와 타격 팔이 순서대로 따라오는지 확인한다.
- 타격 팔은 빠르게 뻗지만 반대 팔은 가드를 유지하고, 머리가 몸통 회전에 과하게 끌려가지 않는지 확인한다.
- 펀치와 스웨이의 최대 회전 구간에서도 복부와 반바지 사이에 틈이 생겨 상체와 하체가 분리되어 보이지 않는지 확인한다.
- 팔을 굽히고 뻗을 때 팔꿈치 연결이 끊기거나 위팔과 아래팔이 겹쳐 깨져 보이지 않는지 확인한다.
- 전진 펀치, 후퇴 펀치, 스웨이 직후 카운터의 준비와 회복 모션이 구분되는지 확인한다.
- 펀치 회복 끝부분에 누른 다음 펀치가 버퍼링되어 자연스럽게 연결되는지 확인한다.
- 사거리 밖에서 펀치하면 체력이 줄지 않는지 확인한다.
- 가까운 거리에서는 펀치 활성 구간과 타격 연출이 일치하는지 확인한다.
- 스트레이트는 짧고 수평으로, 스매시는 큰 회전으로, 어퍼컷은 위로 들리는 피격 반응이 나오는지 확인한다.
- 적중 순간에 짧은 정지감과 공격 방향의 화면 충격이 있고 조작을 끊을 만큼 길지 않은지 확인한다.
- 헛친 펀치는 몸이 조금 더 쏠렸다가 적중 때보다 무겁게 가드로 돌아오는지 확인한다.
- 공격자도 적중 순간 어깨와 몸통이 짧게 더 따라가며 피격자만 혼자 움직이는 인상이 없는지 확인한다.
- 글러브가 상대 몸에 닿기 전에 지나치게 큰 빈 공간을 두고 타격 판정이 발생하지 않는지 확인한다.
- 리치 보너스가 있는 LEO도 글러브와 상대 몸통 사이에 눈에 띄는 빈 공간이 남은 상태에서는 적중하지 않아야 한다. 리치 배율은 상대 몸통 반경을 키우지 않는다.
- 후퇴 펀치는 짧고, 전진 펀치와 카운터는 모션만큼 조금 더 멀리 닿는지 확인한다.
- 앞뒤 깊이 차이가 큰 경우 화면상 거리에 맞게 빗나가는지 확인한다.

### 스웨이와 카운터

- 상대 축의 좌우로 입력하며 스웨이하면 각각 좌우 슬립이 나오는지 확인한다.
- 먼 포스트의 작은 선수도 가드 호흡과 구분될 만큼 머리와 상체가 충분히 이동하는지 확인한다.
- 스웨이는 초반에 빠르게 최대 회피 위치로 들어가고 짧게 유지한 뒤, 진입보다 긴 시간에 걸쳐 가드 자세로 복귀하는지 확인한다.
- 좌우 슬립에서 상체가 이동하는 쪽 반대로 골반과 무릎이 균형을 잡고 몸통이 분리되어 보이지 않는지 확인한다.
- 상대에게서 멀어지는 입력과 방향 입력이 없을 때 뒤로 젖히는 풀백이 나오는지 확인한다.
- 상대 쪽으로 이동하며 스웨이하면 회피에 실패하고 공격에 맞는지 확인한다.
- 스웨이 버튼을 누른 뒤 스틱 방향을 바꿔도 처음 선택한 동작이 끝까지 유지되는지 확인한다.
- 상대가 대각선에 있거나 선수가 회전 중이어도 스웨이의 화면상 이동 방향은 스틱 방향과 일치하는지 확인한다. 회피 성공 여부만 상대 기준으로 별도 판정되어야 한다.
- 스웨이 자세가 만들어진 뒤 `PUNCH`를 누르면 스웨이 종료를 기다리지 않고 현재 자세에서 펀치 준비로 전환되는지 확인한다.
- `SWAY`와 `PUNCH`가 거의 동시에 입력되어도 매번 스웨이 후 펀치 순서로 일관되게 처리되는지 확인한다.
- 스웨이 중 피격되면 미리 입력한 펀치가 피격 회복 뒤에 뒤늦게 실행되지 않는지 확인한다.
- 스웨이 중에는 발이 미끄러지지 않고 제자리에 고정되는지 확인한다.
- 스웨이에서 상체와 골반이 반대 방향으로 균형을 잡고 머리만 따로 꺾여 보이지 않는지 확인한다.
- 기본 가드에서 양쪽 팔꿈치와 글러브가 몸통 바깥으로 벌어지지 않고 얼굴과 몸의 중심선을 보호하도록 안쪽으로 모이는지 확인한다.
- 스웨이를 너무 일찍 또는 늦게 사용하면 방향이 맞아도 공격에 맞는지 확인한다.
- 상대 펀치 타이밍에 스웨이하면 `COUNTER READY`가 표시되는지 확인한다.
- 제한 시간 안의 반격만 강한 카운터가 되는지 확인한다.

### 방향과 원근

- 대각선 시작 위치에서 두 선수가 서로 바라보는지 확인한다.
- 쿼터 뷰에서도 스틱의 상·하·좌·우와 실제 화면 이동 방향이 일치하는지 확인한다.
- 링 일부가 화면 밖으로 이어져 경기장이 충분히 넓게 느껴지는지 확인한다.
- 플레이어가 데드존 안에서 움직일 때 카메라가 불필요하게 따라오지 않는지 확인한다.
- 데드존을 벗어나면 플레이어 쪽에 더 무게를 둔 카메라가 부드럽게 따라오는지 확인한다.
- 경기장 카메라가 움직여도 체력바와 조작부가 화면에 고정되는지 확인한다.
- 카메라 원근과 관계없이 같은 링 거리에서는 펀치 판정이 일관적인지 확인한다.
- 한 선수가 다른 선수의 상하좌우를 돌 때 몸 방향이 자연스럽게 변하는지 확인한다.
- 정면에 가까운 상황에서 캐릭터가 좌우로 빠르게 뒤집히지 않는지 확인한다.
- 두 선수가 `/` 또는 `\` 대각선 구도로 마주 볼 때 몸통이 홀쭉하게 압축되지 않는지 확인한다.
- 두 선수가 가까워지거나 링 가장자리에 몰려도 몸이 겹치거나 팔다리 깊이 순서가 떨리지 않는지 확인한다.
- 먼 선수의 크기와 그림자가 함께 작아지는지 확인한다.
- 매트 패널과 봉제선이 쿼터 뷰의 두 이동 축을 따라가며 카메라가 이동해도 캐릭터 발 아래에서 미끄러져 보이지 않는지 확인한다.
- 접촉 그림자는 발 바로 아래에 남고, 캐릭터보다 진하고 좁게 보여 바닥 높이를 명확히 전달하는지 확인한다.
- 관중석과 조명은 캐릭터 실루엣보다 채도와 대비가 낮아 전투 가독성을 방해하지 않는지 확인한다.

### KO와 재시작

- 체력이 0이 되면 패배 선수가 쓰러지는지 확인한다.
- KO 후 이동, 펀치, 스웨이가 동작하지 않는지 확인한다.
- 배경을 탭해도 즉시 재시작되지 않는지 확인한다.
- `다시 하기` 버튼으로만 체력과 위치가 초기화되는지 확인한다.

## 실제 기기 확인 항목

- 가로 방향이 안정적으로 유지되는가
- 노치와 홈 인디케이터가 조작부를 가리지 않는가
- D-pad와 버튼을 동시에 누르기 편한가
- 일반 타격, 카운터, 스웨이 햅틱이 구분되는가
- 장시간 플레이 시 프레임 저하나 발열이 과도하지 않은가

## 근거리 로비 실기기 테스트

근거리 검색은 iPhone 두 대에서 최종 확인한다. 두 기기에 같은 브랜치 빌드를 설치하고 Wi-Fi와 Bluetooth를 켠 뒤 다음 순서로 테스트한다.

1. 두 기기 모두 첫 실행 시 로컬 네트워크 접근을 허용한다.
2. 첫 번째 기기에서 `NEARBY → 방 만들기`를 누른다.
3. 두 번째 기기에서 `NEARBY → 주변 방 찾기`를 누르고 첫 번째 기기 이름의 방에 입장한다.
4. 양쪽에 서로의 기기 이름과 선택 선수가 표시되는지 확인한다.
5. 한쪽에서 JIN, MASON, LEO를 차례로 선택할 때 상대 화면이 즉시 갱신되는지 확인한다.
6. 양쪽이 준비하면 같은 선수 구성으로 전투 화면에 진입하고 3초 카운트다운이 한 번만 실행되는지 확인한다.
7. 준비 후 선수를 바꾸려면 먼저 준비를 취소해야 하고, 선수 변경 뒤 준비 상태가 남지 않는지 확인한다.
8. 양쪽에서 이동, 펀치와 스웨이가 상대 화면에 전달되고 장시간 이동 후에도 선수 위치와 체력바가 크게 어긋나지 않는지 확인한다.
9. KO 뒤 어느 쪽이든 재대결을 요청할 수 있고 상대 화면에 `재대결 수락`이 표시되는지 확인한다.
10. 요청한 선수가 수락 전 취소하면 상대 화면의 요청이 사라지는지 확인한다.
11. 한 명만 요청한 상태에서는 경기가 초기화되지 않고, 양쪽이 모두 수락한 뒤에만 함께 초기화되어 3초 카운트다운이 시작되는지 확인한다.
12. 한쪽에서 대전을 종료하면 다른 쪽 연결도 종료되고 다시 방 만들기 또는 검색을 시작할 수 있는지 확인한다.

실기기에서는 Wi-Fi 혼잡 상황에서 입력 체감 지연, 위치 보정으로 인한 순간 이동, 발열과 배터리 사용량을 추가 확인한다.
